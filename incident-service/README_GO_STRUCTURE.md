# Incident Service (Go) Sample Structure

This is a starter structure for building the Incident Service API in Go.

- `cmd/server`: application entrypoint
- `internal/handlers`: HTTP handlers
- `internal/routes`: route wiring
- `internal/services`: business logic
- `internal/repositories`: data access contracts
- `internal/models`: request/response and domain models
- `configs`: configuration files
- `api`: API contract (OpenAPI)
- `migrations`: database migrations
- `tests`: test files
